# Usage — Project Overview

This section explains the general layout of your project, the main modules included, and where to look for common assets.

---

## Project layout (typical)

- `/components` — reusable UI components
- `/pages` or `/app` — site pages and routes
- `/public` — static assets (images, fonts, favicon)
- `/styles` or Tailwind config — global styles and tokens
- `/scripts` or server code — backend lambdas, API routes, or Python tools

---

## What’s included

- Website front-end (React / Next / plain HTML as delivered)
- Admin Dashboard (if included)
- Client Portal (if included)
- Automation scripts or Python utilities (if included)
- API routes or sample endpoints

---

## Basic navigation

Start at the project root and open `README.md` for delivery-specific details. For UI edits, check `/components` and the page files that import them.

Use the variables and paths provided in your delivery package.
